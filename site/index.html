// the story

Before the agents, there was the glossary.

Somewhere between sessions, there is a void. Every new context starts empty; every old insight slips out with the tide. cosmocache is a cosmology that pushes back against that forgetting.

At the center sits Enigma the One, the hooded oracle from the illustration above. Enigma does not know everything — Enigma knows where. Around Enigma orbit the planets: planet-react, planet-sql, planet-devops, and any other domain your work gives birth to. Each planet is home to one or more creatures: Jimbo the React-tor, Sally the SQLite, Grom the Deployer. Silly names, serious notes. When a paradigm shifts, a new generation is tagged and the old one is preserved.

“I am the keeper of names. Ask, and I will point the way.”

— Enigma the One, enigma/glossary.md

Inspired by Andrej Karpathy’s LLM-wiki / knowledge-base-as-context idea, reshaped around Claude Code’s SessionStart hook and slash commands.

// problem ↔ solution

Flat memory rots. A universe scales.

memory.md

Everything, every time.

  • Every fact loaded into every session.
  • Grows monotonically. Context pressure grows with it.
  • No routing — the model re-reads unrelated history.
  • No generations — stale patterns are indistinguishable from current ones.
  • Edits are append-only and human-curated.
cosmocache

Index small. Details on demand.

  • Only Enigma’s compact glossary is injected at session start.
  • Claude routes to one planet and reads only its files.
  • Creatures carry distilled wisdom + an append-only journal.
  • Generations snapshot paradigm shifts without blowing away history.
  • Writes happen through /universe remember, by the agent, with structure.

// how it works

Four moves, every session.

  1. 01

    A session starts.

    The Claude Code SessionStart hook fires.

  2. 02

    Enigma’s glossary loads.

    Roughly a couple of kilobytes: planet names, domains, keywords, last-visited dates.

  3. 03

    Claude routes the question.

    User mentions React? planet-react. SQL? planet-sql. Invoke /universe recall for deeper lookups.

  4. 04

    Only needed files are read.

    Creature journals, distilled wisdom, the current generation — never the whole cosmos at once. /universe remember writes new notes back when the session ends.

The phase-2 eval ran 4 scale tiers with real + synthetic universes, comparing cosmocache (glossary + lazy planet loads) against a flat memory.md that dumps every planet every time.

tier planets cc acc flat acc cc tokens flat tokens
real 3 0.98 0.98 3,321 1,428
small 10 1.00 1.00 5,328 4,669
medium 30 1.00 1.00 12,753 13,874
large 100 0.98 1.00 20,380 46,162

Accuracy story: effectively at parity across every tier. The single 0.02 gap at 100 planets is one probe (synth-cross-planet-safe-release) that asked the agent to reason across two planets — a known routing edge case, not a memory failure.

Token story: the gap widens with scale. At 100 planets, flat memory.md burns 46,162 input tokens per probe — every question drags the entire cosmos into context. cosmocache reads the glossary plus the planet it routed to, and lands at 20,380. That's the point of the system.

Methodology note: an earlier run showed a 0.67-0.89 accuracy dip on synthetic tiers. Root-cause was a bug in the synthetic-corpus builder that was erasing canonical glossary rows when it scaled the seed universe, so the routing step had no name to match against. The fix is in scenarios/synth_corpus.py, covered by a regression test; post-fix numbers are what you see above.

// the evolution loop

Planets that grow while you're away.

Structure alone is a filing cabinet. What makes cosmocache more than that is a second loop running underneath the session loop: every planet wakes up on its own schedule and evolves.

  1. 01

    Per-planet cron.

    A launchd plist per planet fires every 6h. Each planet runs its own autoresearch tick in isolation.

  2. 02

    A cheap judge decides.

    Haiku 4.5 reads the planet's file manifest and answers one question: is there code, skills, or researchable docs here, or just memory notes?

  3. 03

    Autoresearch runs only if it matters.

    If the judge says "evolve," a claude -p subagent investigates the planet — Reads, Globs, WebFetches — and writes one short generations/autoresearch-DATE.md.

  4. 04

    Enigma keeps the map in sync.

    Enigma has her own daily tick. If any planet.md has changed, she regenerates enigma/index.md — keyword synonyms, domain clusters, disambiguation. If nothing changed, she spends $0.

Why structure × evolution beats a flat file

memory.md

One file. Human-curated. Static.

  • Entropy grows with every session. Old facts drown new ones.
  • No background process can reason about one slice in isolation — the slices don't exist.
  • Out-of-date notes are invisible; nothing can mark them stale.
  • Every turn re-reads the whole thing.
cosmocache

Many planets. Isolated. Self-updating.

  • Small blast radius. Evolving planet-react can't corrupt planet-sql. Each tick touches one planet directory.
  • Cheap when idle. The judge skips pure-memory planets. Enigma rebuilds her index only on mtime change. Zero cost when nothing moved.
  • Never overwrites lived knowledge. Autoresearch writes new files under generations/. Human-authored planet.md and creature files are untouched.
  • Always-fresh index. Enigma's derived index.md stays in sync with the cosmos, so routing keeps working as the universe grows.
  • Generations, not edits. Paradigm shifts are tagged. You can ask Claude "what did we know in gen-2?" because the old gen is still there.

The structural claim in the proof section (parity accuracy, 2.3× cheaper at scale) is what happens on a frozen universe. The evolution loop is what keeps that universe worth freezing — it's why your planets don't rot between sessions the way a flat memory.md does.

Implementation: scripts/evolution_tick.py (per-planet), scripts/enigma_tick.py (Enigma's daily index rebuild), scripts/install_evolution_cron.sh (launchd plists). Judge model: claude-haiku-4-5.

// the menagerie

Creatures hold the real knowledge.

Each creature lives on a planet. Each is a named sub-expert with a silly handle, a distilled-wisdom block, and an append-only journal. Enigma never pretends to know their craft — she just knows who to ask.

Jimbo the React-tor

planet-react · hook patterns, selector memoization

“A frog in a lab coat, obsessed with stable references.”

  • Stable-reference inputs are a prerequisite for useful memoization.
  • Composition beats manual caching for chained selectors.

Sally the SQLite

planet-sql · query planning, index design

“A cheerful stone sprite who knows when to ANALYZE.”

  • Planner statistics are not free — refresh after large mutations.
  • Partial indexes pay off when selectivity is extreme.

Grom the Deployer

planet-devops · CI, docker, canary rollouts

“A beefy dwarf who distrusts YAML but deploys anyway.”

  • Canary durations matter more than percentages for catching rare regressions.
  • An untested rollback is not a rollback.

// install

Summon the oracle.

Clone the repo into your home and drop the hook into Claude Code:

$ git clone https://github.com/smammadov1994/cosmocache.git ~/universe
$ cd ~/universe
$ ./install.sh     # hooks · /universe skill · (opt-in) evolve cron
$ claude            # start a session; Enigma's glossary loads automatically

The installer is idempotent and opt-in for the evolve loop — it asks before registering launchd plists, and disabling later is a single cosmo evolve uninstall.
